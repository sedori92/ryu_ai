# 03_feature_spec.md

## 기능 1. 텍스트 입력
사용자는 textarea에 감성 분석할 문장을 입력한다.

### 제약조건
- 빈 문자열은 허용하지 않는다.
- 공백만 있는 문자열은 빈 입력으로 처리한다.
- 최대 글자 수는 1,000자로 제한한다.

### 완료 기준
- 사용자가 문장을 입력할 수 있다.
- 1,000자를 초과하면 안내 메시지를 표시한다.

## 기능 2. 감성 분석 요청
사용자가 분석 버튼을 클릭하면 프론트엔드가 `/api/analyze`로 POST 요청을 보낸다.

### 요청 데이터
```json
{
  "text": "오늘은 기분이 좋습니다."
}
```

### 완료 기준
- 빈 입력이면 API를 호출하지 않는다.
- 정상 입력이면 API를 호출한다.

## 기능 3. 결과 표시
결과는 다음 항목으로 표시한다.

- 감성 결과: positive, negative, neutral
- 한글 표시: 긍정, 부정, 중립
- 신뢰도: 0~100
- 분석 이유

## 기능 4. 오류 메시지
다음 상황에서 오류 메시지를 표시한다.

- 입력값 없음
- 글자 수 초과
- 네트워크 오류
- OpenAI API 오류
- 서버 내부 오류

## 기능 5. 분석 이력 저장
분석 완료 후 Supabase에 다음 정보를 저장한다.

- 입력 텍스트
- 감성 결과
- 신뢰도
- 분석 이유
- 생성 시각
